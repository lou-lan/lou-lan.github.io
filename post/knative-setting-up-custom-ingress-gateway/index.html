Knative 自定义 Ingress 网关

Knative 使用共享的 Ingress 网关为 Knative 服务网格内的所有传入流量提供服务，该服务网格是knative-serving 命名空间下的 knative-ingress-gateway 网关。默认情况下，我们在 istio-system 名称空间下使用 Istio 网关服务 istio-ingressgateway 作为其基础服务。可以按以下方式用自己的服务替换该服务。

步骤1：创建网关服务和部署实例

需要首先创建网关服务和部署实例来处理流量。最简单的方式应该在模板中，Istio release。

            - "45s" #drainDuration

            - "1m0s" #parentShutdownDuration

            - zipkin:9411

            - --proxyAdminPort
            - "15000"

            - NONE
            - --discoveryAddress
            - istio-pilot:8080
          resources:
            requests:
              cpu: 10m
          env:
            - name: POD_NAME
              valueFrom:
                fieldRef:
                  apiVersion: v1
                  fieldPath: metadata.name
            - name: POD_NAMESPACE
              valueFrom:

            - name: INSTANCE_IP

                fieldRef:

步骤2：更新 Knative Gateway

在 knative-serving namespace 下更新网关实例 knative-ingress-gateway：

1

kubectl edit gateway knative-ingress-gateway -n knative-serving

用你的服务 label 替换 label selector：

1

istio: ingressgateway

对于上述服务，应当更新如下：

1

custom: ingressgateway

如果服务端口发生了变化（与 istio-ingressgateway 相比），请相应地更新网关中的端口信息。

步骤3：更新 Gateway Configmap

在 knative-serving namespace 下更新网关 configmap config-istio：

1

kubectl edit configmap config-istio -n knative-serving

将 ingress-gateway 字段替换为服务的标准网址。对于上述服务，应当更新如下：

1

custom-ingressgateway.istio-system.svc.cluster.local
